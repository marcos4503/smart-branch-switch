# 🧹 Periodic Clean of `gh-pages` History

This file contains instructions to periodically reset the history of this Branch, to prevent it from unnecessarily bloating the Repository due to old files it once contained. This bloating causes the Repository to grow unnecessarily, which can be a nuisance when cloning the Repository or keeping it on the PC. To recreate Branch `gh-pages`, follow the steps below, in your Local Repository on PC:

1. Using GitHub Desktop or Git, switch the currently active Branch to `gh-pages` Branch.
2. Copy all the files from the Branch to a location on your PC.
3. Using GitHub Desktop or Git, switch the currently active Branch to `main` Branch.
4. Open Git Bash in the root of your Local Repository.
5. Execute the following commands to delete the `gh-pages` Branch. This will delete it and also delete all its file history, both in the Local Repository and the Remote Repository.
     - `git branch -D gh-pages`
     - `git push origin --delete gh-pages`
6. Still in Git Bash, run the command below to create a new, orphaned `gh-pages` Branch with a 100% empty history.
     - `git checkout --orphan gh-pages`
7. After executing the command in step `6`, Git creates a new orphaned `gh-pages` Branch. It will also change the currently active Branch in the Local Repository to `gh-pages`, but since the new Branch is orphaned, it will keep the files from the last Branch in the Local Repository's Work Directory. Confirm that the current Branch active is `gh-pages`. If it is, use the command below to clean up these files before we continue.
    - `git rm -rf .`
8. Now you have two options:
    - **Bring back ALL the files copied in step `2`, all at once:**
      - Copy back to Branch `gh-pages`, all the files you copied in step `2`.
      - Execute the following commands in Git Bash:
        - `git add -A`
        - `git commit -m "Reset gh-pages with fresh files"`
        - `git push -f origin gh-pages`
    - **Bring back a SOME of the files copied in step `2`, a little at a time:**
      - **8.1 -** Copy back to Branch `gh-pages` some of the files you copied in step `2`.
      - **8.2 -** Execute the following commands in Git Bash:
        - `git add -A`
        - `git commit -m "Reset gh-pages with fresh files"`
        - `git push -f origin gh-pages`
      - **8.3 -** Go back to step `8.1`. Repeat this cycle until you have brought back to Branch `gh-pages` all the files you copied in step `2`.
9. Run the command below to configure the Branch Upstream in the Local Repository. This will link Branch `gh-pages` in the Remote Repository and Branch `gh-pages` in the Local Repository.
   - `git push -u origin gh-pages`
10. Now, run the command below. This command will cause Git to delete all files from the Local Repository that were abandoned when the old `gh-pages` Branch was deleted to make way for the new `gh-pages` Branch.
    - `git gc --prune=now --aggressive` 
11. That's all. The Branch `gh-pages` has now been recreated with a completely clean history. You can now close Git Bash and go back to managing your Local Repository as usual.

> [!NOTE]
> As a final check, remember to access the Repository on the GitHub Website and make sure that `GitHub Pages` is indeed still active for `gh-pages` Branch.
